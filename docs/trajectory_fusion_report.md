# 雷达与AIS轨迹融合逻辑分析报告

本报告对 [TrajectoryFusion.m](file:///c:/Users/fairyland/Desktop/ais-perception-fusion/TrajectoryFusion.m) 脚本中实现的雷达与船舶自动识别系统（AIS）轨迹融合逻辑进行了详细分解和技术分析。该脚本的主要任务是将匹配后的异构传感器（雷达与AIS）数据在时空上对齐，并通过加权算法合成出一条更平滑、准确的连续轨迹。

## 1. 核心处理流程概述

整个融合算法流程可以分为以下几个关键步骤：
1. **数据准备与清洗**：基于匹配结果提取成对轨迹，并处理异常点。
2. **时间轴同步（Time Synchronization）**：求取时间交集并建立高频固定的统一时间轴。
3. **空间插值对齐（Spatial Interpolation）**：将不同采样率的两组轨迹补全至统一时间轴上。
4. **加权融合（Weighted Fusion）**：基于设定的权重和对齐后的点位求融合位置。
5. **结果存储与可视化**：将融合轨迹按结构体存储并绘图验证。

## 2. 逻辑细节深度解析

### 2.1 参数配置与初始化
脚本顶层定义了融合工程的核心参数：
- **采样周期 (`resample_dt`)**：统一设置为 `1秒`。这对低频的AIS数据进行了上采样，同时统一了雷达和AIS的时间基准。
- **融合权重 (`weight_radar` / `weight_ais`)**：当前采用**常数加权平均法**，雷达和AIS各占 `0.5` 的权重。这是一种基础且有效的基线做法（代码注释中指出未来可升级为卡尔曼滤波等动态协方差算法）。

### 2.2 数据预处理与清洗
在进入插值和融合前，系统对各个实体（UID/MMSI）的数据进行了严格的清洗：
- **去重处理**：使用 `unique(time)` 剔除了时间戳重复出现的脏数据。这一步非常关键，因为MATLAB的 `interp1` 插值函数要求自变量必须是**严格单调递增**的，遇到重复时间戳会导致报错。
- **数据量校验**：拦截并过滤掉有效轨迹点少于2个的情况。

### 2.3 时空重合区间判断
融合仅在两组传感器同时捕获到目标的时段内进行：
- **求时间交集**：
  ```matlab
  start_time = max(min(r_time), min(a_time));
  end_time = min(max(r_time), max(a_time));
  ```
  只有当两者同时存在观察值，且 `start_time < end_time` 时，才会生成固定频率的 `sync_time`。

### 2.4 插值采样 (pchip 保形插值)
为了解决雷达与AIS采样时刻不一致、频率不一致的问题，代码进行了插值处理。
- **时间格式转换**：引入 `datenum` 将高层时间对象转换为底层数值类型，防止部分早期甚至新版MATLAB在插值运算时报错或降速。
- **插值算法选择**：选用了 `'pchip'`（保形分段三次插值，Shape-preserving piecewise cubic interpolation）。
  - **优势**：不同于传统的 `'spline'`（三次样条插值），`pchip` 避免了在数据突变或间隔较大时产生**“过冲（Overshoot）”**现象。它能较好地保留真实物理世界中船舶运动轨迹的走势，不会在两次观测点之间产生不符合物理常理“甩尾”和震荡。

### 2.5 融合计算
经过前置处理，雷达与AIS数据已转化为长度相同、时刻相同的两组经纬度序列：`_lon_interp` 与 `_lat_interp`。
- 直接采用线性加权法进行融合：
  ```matlab
  fused_lon = weight_radar * r_lon_interp + weight_ais * a_lon_interp;
  fused_lat = weight_radar * r_lat_interp + weight_ais * a_lat_interp;
  ```

### 2.6 输出与可视化
- **结构化存储**：最终融合的结果通过不断追加填充至预设的 `fused_trajectories` 结构体数组中。该结构不仅保存了融合坐标，而且还完好保留了插值对齐后的原始对应坐标，便于后续进行误差分析或评估。
- **二维可视化**：对于选定的航迹索引，生成经纬度二维曲线：
  - 用虚线与点划线绘制重叠区间的雷达原始轨迹与AIS原始轨迹。
  - 用醒目的实线绘制融合轨迹，并标记首末节点，直观体现对齐和混合后的平滑效果。

## 3. 总结与改进建议

### 优点：
1. **鲁棒性强**：在预处理流程中进行了充足的过滤（有效点数量验证、重复时间清洗、时间交集验证），保障了批处理执行的稳定性。
2. **插值算法合理**：针对船舶运动这一物理过程选择了 `pchip`，极大改善了重采样产生的伪影与失真。
3. **代码结构清晰**：数据提取、对齐、融合分步进行，可扩展性很好。

### 未来可优化方向（进阶建议）：
1. **动态权重 / 滤波算法**：由于雷达可能产生短暂的漂移、而AIS在长时间可能有延迟，常数权重 `0.5:0.5` 略显单薄。可以基于协方差矩阵引入**卡尔曼滤波 (Kalman Filter)** 或者根据两个高频点/低频点之间的时间跳变差额设定**动态权重**。
2. **时间戳精度**：目前采样粒度设定为 1 秒 (`seconds(1)`)，如果在极端高频的近距离微观避碰场景，可以考虑提高至 `0.1s`。
3. **坐标投影**：目前代码直接在线性的经纬度上做加权平均和插值。如果研究范围内跨越较大经纬度，受地球曲率影响会导致失真。严谨的做法是**先转换为墨卡托坐标（UTM / X-Y平面距离）**，在欧式空间进行插值融合，最后再逆投影回经纬度。
